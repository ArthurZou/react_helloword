# NeuralCanvas

A diffusion model pipeline that generates production-ready UI mockups from natural language
descriptions. Designers describe a screen; the model renders it. Engineers iterate from there.

---

## Architecture

| Component | Technology |
|---|---|
| Backbone | Stable Diffusion XL 1.0 |
| Fine-tuning | LoRA on 12k design-system screenshots |
| Conditioning | CLIP text encoder + layout embeddings |
| Post-processing | SAM segmentation + layer extraction |
| API | FastAPI + async task queue (Celery + Redis) |
| Frontend | React + canvas renderer |

---

## Pipeline Overview

```python
from neuralcanvas import Pipeline

pipe = Pipeline.from_pretrained("neuralcanvas/sdxl-ui-lora-v2")

result = pipe(
    prompt="A dark-mode dashboard with a sidebar nav, "
           "a metrics grid, and a data table below",
    negative_prompt="blurry, low resolution, watermark",
    num_inference_steps=30,
    guidance_scale=7.5,
)

result.save("mockup.png")
result.export_layers("mockup_layers/")  # PSD-compatible layer export
```

## Training Setup

```bash
# prepare dataset
python scripts/prepare_dataset.py \
  --source ./screenshots \
  --output ./data/processed \
  --resolution 1024

# run LoRA fine-tune (requires 2x A100)
accelerate launch train_lora.py \
  --base_model stabilityai/stable-diffusion-xl-base-1.0 \
  --dataset ./data/processed \
  --rank 64 \
  --lr 1e-4 \
  --epochs 20 \
  --output ./checkpoints/v2
```

---

## Lesson Learned

> The hardest part wasn't the model — it was the **data pipeline**. Consistent labeling of UI
> components at scale required three rounds of annotation tooling before the quality was
> acceptable for fine-tuning. **Invest in your data before you invest in your architecture.**
